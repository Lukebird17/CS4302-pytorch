📄 核心文件说明
---------------

【CUDA算子实现】
custom_ops/fused_ops.cu
  - 包含7项优化的完整CUDA实现
  - 优化1: 融合LayerNorm+Residual+Dropout (eval)
  - 优化2: Welford一遍扫描LayerNorm
  - 优化3: 针对hidden_size=768特化
  - 优化4: 快速GELU (tanh近似)
  - 优化5: Online Softmax
  - 优化6: float4向量化
  - 优化7: Bias+GELU融合

custom_ops/fused_ops_wrapper.py
  - Python接口封装
  - 提供fallback机制（编译失败时降级到PyTorch实现）

custom_ops/setup.py
  - PyTorch C++扩展编译脚本
  - 配置NVCC编译选项

【模型优化】
models/bert_optimized.py
  - 创建优化版BERT模型
  - 替换BertSelfOutput和BertOutput为融合实现
  - 替换GELUActivation为快速GELU

【测试脚本】
test_performance.py
  - 干净的性能对比测试（无hooks）
  - 测试6个batch size: 1, 4, 8, 16, 32, 64
  - 每个配置5轮测试，每轮100次迭代
  - 计算均值、标准差、变异系数

visualize_results.py
  - 生成4张性能对比图表:
    * 延迟对比柱状图
    * 加速比图
    * 性能提升百分比
    * 优化分解表

【便捷脚本】
quick_test.sh
  - 一键测试脚本
  - 自动检查CUDA算子编译状态
  - 运行性能测试

run_test.sh
  - 原完整测试流程
  - 包含环境检查、编译、测试等

🚀 使用流程
-----------

1. 【首次使用】编译CUDA算子
   cd custom_ops
   python setup.py install
   cd ..

2. 【运行测试】
   方式1: 快速测试
   ./quick_test.sh

   方式2: 手动测试
   python test_performance.py

3. 【生成图表】
   python visualize_results.py

4. 【查看结果】
   - 终端输出: 实时性能数据
   - 图表: performance_comparison.png


💡 开发说明
-----------

【修改CUDA算子】
1. 编辑 custom_ops/fused_ops.cu
2. 重新编译: cd custom_ops && rm -rf build && python setup.py install && cd ..
3. 运行测试验证

【添加新优化】
1. 在 fused_ops.cu 中实现CUDA kernel
2. 在 fused_ops_wrapper.py 中添加Python接口
3. 在 bert_optimized.py 中替换对应模块
4. 测试性能

【修改测试配置】
编辑 test_performance.py 中的 TEST_CONFIGS:
  - 修改batch size
  - 修改sequence length
  - 调整测试轮数和迭代次数


⚠️ 注意事项
-----------

1. 需要CUDA支持的GPU
2. 需要PyTorch 2.1.0（带CUDA）
3. 编译前确保nvcc可用
4. 首次编译可能需要5-10分钟
5. batch size过大可能OOM（显存不足）


🔧 故障排除
-----------

【编译失败】
- 检查CUDA版本: nvcc --version
- 检查PyTorch版本: python -c "import torch; print(torch.__version__)"
- 清理build: cd custom_ops && rm -rf build && cd ..

【性能无提升】
- 确认使用了优化模型: 检查test_performance.py导入
- 检查算子是否加载: python -c "import bert_fused_ops; print('OK')"
- 查看是否fallback到PyTorch实现

【OOM错误】
- 减小batch size
- 减少测试轮数
- 使用更大显存的GPU
