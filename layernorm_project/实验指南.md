# LayerNorm CUDA算子调研实验指南

## 实验目标

本实验旨在深入理解PyTorch中LayerNorm算子的CUDA实现，并通过自定义优化实现来提升性能。

## 实验步骤详解

### 第一步：环境检查

运行安装测试脚本，确保环境配置正确：

```bash
python test_installation.py
```

**预期输出**：
```
✓ PyTorch 已安装
✓ NumPy 已安装
✓ Transformers 已安装
✓ CUDA可用
✓ GPU名称: NVIDIA XXX
```

**如果出现问题**：
- CUDA不可用：检查驱动和CUDA toolkit
- 包未安装：运行 `pip install -r requirements.txt`
- GPU不可用：需要有NVIDIA GPU的机器

### 第二步：LayerNorm算子调研

运行调研脚本，分析LayerNorm的实现：

```bash
python layernorm_research.py
```

**这一步会做什么**：

1. **PyTorch Profiler分析**
   - 使用Profiler记录LayerNorm的执行
   - 统计CPU和CUDA时间
   - 记录内存使用情况

2. **CUDA Kernel分析**
   - 识别LayerNorm调用的kernel
   - 统计kernel调用次数
   - 分析kernel执行时间

3. **实现原理分析**
   - 打印LayerNorm的数学公式
   - 说明计算步骤
   - 展示PyTorch源码路径

4. **性能基准测试**
   - 测试不同配置下的性能
   - 包括不同batch size、sequence length

5. **内存访问分析**
   - 分析内存布局
   - 说明访问模式
   - 提出优化策略

6. **CUDA Runtime分析**
   - 列出使用的CUDA API
   - 说明调用时机和作用

**输出文件**：
- `profiler_results/layernorm_trace.json` - 可在chrome://tracing查看
- `profiler_results/layernorm_stats.txt` - 详细统计
- `layernorm_research_report.txt` - 调研报告

**关键观察点**：

1. **查看Profiler结果**
   ```bash
   # 在Chrome浏览器中打开 chrome://tracing
   # 加载 layernorm_trace.json
   # 观察：
   # - LayerNorm kernel的执行时间
   # - 与其他操作的时间对比
   # - 内存操作的时间
   ```

2. **分析kernel统计**
   - LayerNorm相关的kernel有哪些
   - 调用频率如何
   - 哪个kernel最耗时

3. **理解实现路径**
   ```
   Python: torch.nn.LayerNorm
     ↓
   C++: at::layer_norm
     ↓
   CUDA: LayerNormForwardKernel
   ```

### 第三步：BERT推理评测

在实际的BERT模型上测试LayerNorm性能：

```bash
python bert_inference_benchmark.py
```

**这一步会做什么**：

1. **加载BERT模型**
   - 使用bert-base-uncased
   - 统计LayerNorm层数量（应该是24个）

2. **加载IMDB数据集**
   - 从Hugging Face加载测试数据
   - 准备推理样本

3. **性能测试**
   - 测试不同batch size的推理时间
   - 统计吞吐量
   - 记录延迟分布

4. **LayerNorm Profiling**
   - 单独分析LayerNorm的时间占比
   - 生成详细的trace

**输出文件**：
- `benchmark_results.json` - 评测结果
- `profiler_results/bert_layernorm_trace.json` - BERT的trace

**关键观察点**：

1. **LayerNorm占比**
   - 在整个BERT推理中占多少时间
   - 是否值得优化

2. **性能瓶颈**
   - Batch size对性能的影响
   - 是否受限于内存带宽

3. **优化空间**
   - 如果LayerNorm占比高（>5%），值得优化
   - 如果占比低，可能需要优化其他算子

### 第四步：编译自定义CUDA扩展

编译优化的LayerNorm实现：

```bash
python setup.py install
```

**编译过程**：

1. **nvcc编译CUDA代码**
   ```
   custom_layernorm.cu -> custom_layernorm.o
   ```

2. **链接PyTorch库**
   - 链接libtorch
   - 链接CUDA runtime

3. **生成Python扩展**
   ```
   custom_layernorm_cuda.so
   ```

**可能的问题**：

1. **架构不匹配**
   ```bash
   # 查看你的GPU架构
   nvidia-smi --query-gpu=compute_cap --format=csv
   
   # 修改setup.py中的-gencode参数
   # 例如：sm_70 (V100), sm_75 (T4), sm_80 (A100), sm_86 (RTX 3090)
   ```

2. **编译器版本**
   ```bash
   # CUDA 11.0需要gcc 7-9
   # CUDA 11.8需要gcc 7-11
   # 检查版本
   gcc --version
   nvcc --version
   ```

3. **内存不足**
   - 编译可能需要大量内存
   - 建议至少8GB RAM

**验证编译**：

```bash
# 测试是否可以导入
python -c "import custom_layernorm_cuda; print('成功!')"

# 运行单元测试
python custom_layernorm.py
```

**预期输出**：
```
测试自定义LayerNorm...
最大差异: 0.000xxx
平均差异: 0.000xxx
✓ 测试通过: 自定义LayerNorm输出正确!
```

3. **生成性能图表**
   - 延迟对比柱状图

**输出文件**：
- `performance_comparison_results.json` - 详细数据
- `performance_plots/layernorm_performance_comparison.png` - 可视化

**关键指标**：

1. **延迟降低**
   ```
   改进 = (原生时间 - 优化时间) / 原生时间 × 100%
   ```

2. **加速比**
   ```
   加速比 = 原生时间 / 优化时间
   ```

3. **吞吐量提升**
   ```
   吞吐量 = samples_per_second
   ```

## 常见问题解答

### Q1: 编译自定义扩展失败怎么办？

**A**: 检查以下几点：
1. CUDA版本与PyTorch版本是否匹配
2. gcc版本是否符合要求
3. 是否有足够的内存
4. 查看详细错误信息

### Q2: 自定义实现比原生慢？

**A**: 可能的原因：
1. PyTorch的实现已经高度优化
2. 测试配置不合适（太小的batch/hidden）
3. 优化策略不够好
4. GPU架构特性没有充分利用

这是正常的！学习的目的是理解实现，而不是一定要超越高度优化的库。

### Q3: 结果不正确怎么办？

**A**: 检查：
1. 数值精度问题（float32的精度限制）
2. Reduction实现是否正确
3. 内存访问边界
4. 同步问题

### Q4: 显存不足？

**A**: 减小：
1. batch_size
2. sequence_length
3. 测试迭代次数

### Q5: 如何提高加速比？

**A**: 尝试：
1. 更激进的向量化
2. 算子融合
3. 使用FP16
4. 优化特定配置（如hidden_size=768）
