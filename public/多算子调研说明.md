## 概述

这个工具对4个关键CUDA算子进行全面调研：
1. **addmm** - 矩阵乘加操作
2. **softmax** - Softmax归一化
3. **layernorm** - 层归一化
4. **transpose** - 矩阵转置

同时深入分析**CUDA Runtime API**的使用情况。

## 调研内容详解

### 算子1: ADDMM (矩阵乘加)

**数学公式**:
```
out = β * input + α * (mat1 @ mat2)
```

**源码位置**:
```
aten/src/ATen/native/native_functions.yaml  # 声明
aten/src/ATen/native/cuda/Blas.cpp          # CUDA实现
aten/src/ATen/cuda/CUDABlas.cpp             # cuBLAS封装
```

**关键实现**:
- 调用cuBLAS库的 `cublasSgemm` 或 `cublasGemmEx`
- 高度优化，支持Tensor Core
- 使用矩阵分块(Tiling)策略

**并行化**:
- Grid维度: `(M/tile_m, N/tile_n)`
- Block维度: `(tile_size, tile_size)`
- 每个线程处理多个元素

### 算子2: SOFTMAX

**数学公式**:
```
softmax(x_i) = exp(x_i - max(x)) / Σ exp(x_j - max(x))
```

**源码位置**:
```
aten/src/ATen/native/native_functions.yaml  # 声明
aten/src/ATen/native/cuda/SoftMax.cu        # CUDA实现
```

**关键实现**:
- `cunn_SoftMaxForward` - 基础kernel
- `fast_softmax_kernel` - 优化版本
- 在线算法 (Online Algorithm)

**并行化**:
- Grid维度: `(batch_size, seq_len)`
- 每个block处理一行的softmax
- 三阶段: max reduction → exp + sum → normalize

**优化策略**:
- Warp shuffle reduction
- 向量化加载 (float4)
- 融合多个操作减少内存访问

### 算子3: LAYERNORM

**数学公式**:
```
y = γ * (x - μ) / √(σ² + ε) + β
```

**源码位置**:
```
aten/src/ATen/native/native_functions.yaml      # 声明
aten/src/ATen/native/cuda/layer_norm_kernel.cu  # CUDA实现
```

**关键实现**:
- `LayerNormForwardKernel`
- `RowwiseMomentsCUDAKernel`

**并行化**:
- Grid维度: `(batch_size * seq_len, 1, 1)`
- 每个block处理一个token的所有features
- Reduction计算均值和方差

**优化策略**:
- Warp shuffle reduction
- 向量化访问 (float4)
- 共享内存优化

### 算子4: TRANSPOSE

**功能**:
```
交换张量的维度
y[i, j] = x[j, i]
```

**源码位置**:
```
aten/src/ATen/native/native_functions.yaml     # 声明
aten/src/ATen/native/TensorTransformations.cpp # View操作
aten/src/ATen/native/cuda/Copy.cu              # Copy操作
aten/src/ATen/native/cuda/Transpose.cu         # Transpose kernel
```

**两种实现**:

1. **View操作** (推荐)
   - 只改变stride和shape元数据
   - 不移动数据
   - O(1)时间复杂度

2. **Copy操作** (需要时)
   - 实际移动数据
   - 使用 `transpose_copy_kernel`

**并行化 (Copy时)**:
- 使用Shared Memory Transpose
- 避免非合并访问
- Padding避免bank conflict

**关键优化**:
```cuda
__shared__ float tile[TILE_DIM][TILE_DIM + 1];  // +1避免bank conflict
```

### CUDA Runtime API调研

#### 1. 内存管理API

**cudaMalloc / cudaFree**
- 作用: GPU内存分配/释放
- 位置: `c10/cuda/CUDACachingAllocator.cpp`
- 使用: PyTorch通过内存池优化

**cudaMemcpy / cudaMemcpyAsync**
- 作用: CPU-GPU数据传输
- 位置: `c10/cuda/CUDAStream.cpp`
- 类型:
  - `cudaMemcpyHostToDevice` - CPU → GPU
  - `cudaMemcpyDeviceToHost` - GPU → CPU
  - `cudaMemcpyDeviceToDevice` - GPU → GPU

**cudaMallocHost**
- 作用: 分配页锁定内存(pinned memory)
- 优点: 加速CPU-GPU传输
- 位置: `c10/cuda/CUDACachingAllocator.cpp`

#### 2. Kernel启动API

**cudaLaunchKernel**
- 作用: 启动CUDA kernel
- 参数:
  - `func`: kernel函数指针
  - `gridDim`: Grid维度
  - `blockDim`: Block维度
  - `args`: kernel参数
  - `sharedMem`: shared memory大小
  - `stream`: CUDA stream
- 位置: `c10/cuda/CUDAStream.h`

#### 3. 同步API

**cudaDeviceSynchronize**
- 作用: 等待GPU所有操作完成
- 特点: 阻塞CPU
- 位置: `c10/cuda/CUDAFunctions.cpp`
- 使用场景: 计时、调试

**cudaStreamSynchronize**
- 作用: 等待指定stream完成
- 特点: 只等待一个stream
- 位置: `c10/cuda/CUDAStream.cpp`

#### 4. Stream管理API

**cudaStreamCreate / cudaStreamDestroy**
- 作用: Stream创建/销毁
- 用途: 并发执行多个操作
- 位置: `c10/cuda/CUDAStream.cpp`

**Stream的作用**:
- 并发执行: 多个stream可以同时运行
- 异步操作: kernel和memcpy可以异步
- 隐藏延迟: 计算和传输重叠

#### 5. Event管理API

**cudaEventCreate / cudaEventRecord / cudaEventElapsedTime**
- 作用: GPU计时
- 用途: 精确测量kernel执行时间
- 位置: `c10/cuda/CUDAEvent.h`

**使用示例**:
```cpp
cudaEvent_t start, end;
cudaEventCreate(&start);
cudaEventCreate(&end);

cudaEventRecord(start, stream);
// ... kernel execution ...
cudaEventRecord(end, stream);

cudaEventSynchronize(end);
float ms;
cudaEventElapsedTime(&ms, start, end);
```

#### 6. 设备管理API

**cudaGetDeviceCount / cudaSetDevice / cudaGetDevice**
- 作用: GPU数量查询和选择
- 位置: `c10/cuda/CUDAFunctions.cpp`

**cudaGetDeviceProperties**
- 作用: 获取GPU属性
- 返回信息:
  - 名称、计算能力
  - 显存大小
  - SM数量、warp大小
  - Shared memory大小
- 位置: `c10/cuda/CUDAFunctions.cpp`

#### 7. 错误处理API

**cudaGetLastError / cudaPeekAtLastError**
- 作用: 获取错误信息
- PyTorch封装:
  - `C10_CUDA_CHECK(expr)`
  - `TORCH_CUDA_CHECK(expr)`
  - `AT_CUDA_CHECK(expr)`

### 调用时机总结

| 场景 | 调用的API |
|------|----------|
| 创建Tensor | cudaMalloc, cudaGetDevice |
| 数据传输 | cudaMemcpy / cudaMemcpyAsync |
| 执行算子 | cudaLaunchKernel |
| 性能测试 | cudaEventCreate/Record/ElapsedTime, cudaDeviceSynchronize |
| 多GPU训练 | cudaSetDevice, cudaStreamCreate |
| 释放资源 | cudaFree, cudaStreamDestroy |

## 常见问题

### Q: 为什么addmm这么快?
A: 因为使用了高度优化的cuBLAS库，并且可能使用了Tensor Core加速。

### Q: transpose为什么有两个时间?
A: View操作只改变元数据，几乎没有开销；Copy操作需要实际移动数据，有明显开销。

### Q: 如何找到某个算子的CUDA kernel?
A: 
1. 在 `native_functions.yaml` 中找到声明
2. 查看dispatch到CUDA的实现
3. 在对应的.cu文件中找到kernel函数

### Q: CUDA Runtime API在哪里调用?
A: 不是直接调用，而是通过c10库封装，主要在 `c10/cuda/` 目录下。
